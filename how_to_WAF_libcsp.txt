In order to build libcsp, targeting the STM32 F411RE NUCLEO board, follow these steps:

1) Use Ubuntu or WSL; on Windows doesn't work


2) Setup on Linux the ARM GNU toolchain (compilers, linkers, debuggers...) 
Just copy and paste from this guide:
https://lindevs.com/install-arm-gnu-toolchain-on-ubuntu


3) Go to the directory	<ur_workspace>/OBC/Libraries/libcsp


4) Launch the following WAF config command (with ur directories ofc)

./waf configure --toolchain=arm-none-eabi- --with-os=freertos --prefix=/mnt/c/Users/Alessandro/STM32CubeIDE/workspace_1.13.2/OBC/Libraries --includes=/mnt/c/Users/Alessandro/STM32CubeIDE/workspace_1.13.2/OBC/Middlewares/Third_Party/FreeRTOS-Kernel/include,/mnt/c/Users/Alessandro/STM32CubeIDE/workspace_1.13.2/OBC/Core/Inc,/mnt/c/Users/Alessandro/STM32CubeIDE/workspace_1.13.2/OBC/Middlewares/Third_Party/FreeRTOS-Kernel/portable/GCC/ARM_CM4F -v CFLAGS="-mcpu=cortex-m4 -mfloat-abi=hard -mfpu=fpv4-sp-d16"


5) Launch the following WAF build command

./waf build install


6) Pick up the libcsp.a from the \OBC\Libraries\libcsp\build and copy it inside OBC/Libraries


7a) Go to STM32CUBEIDE and inside Project -> Properties -> C/C++ General -> Tool Settings -> MCU GCC Linker ->
	Miscellaneous -> Other flags -> "-Wl,--whole-archive -lcsp -Wl,--no-whole-archive"


7b) Go to STM32CUBEIDE and inside Project -> Properties -> C/C++ General -> Tool Settings -> MCU GCC Linker ->
	Libraries -> Libraries search path (-L) -> "${workspace_loc:/OBC/Libraries}"

8) Build
